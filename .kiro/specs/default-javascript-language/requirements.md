# 需求文档

## 简介

本功能旨在调整智能体系统的提示词配置，确保所有智能体在遇到需要编程解决问题时，默认使用 JavaScript 而不是 Python。这包括更新相关提示词文件，明确指定 JavaScript 作为首选编程语言。

## 术语表

- **智能体系统**: 智能体社会系统，包含根智能体和各级子智能体
- **提示词配置**: 提示词配置文件，位于 config/prompts/ 目录下
- **根智能体**: 根智能体，负责创建和管理子智能体
- **基础提示词**: 基础提示词，所有智能体共享的基础行为规范
- **JavaScript**: 默认编程语言，用于解决编程相关问题

## 需求

### 需求 1: 基础提示词默认语言设置

**用户故事:** 作为智能体系统管理员，我希望基础提示词明确指定 JavaScript 为默认编程语言，以便所有智能体在编程任务中优先使用 JavaScript。

#### 验收标准

1. 当智能体遇到没有明确语言要求的编程任务时，基础提示词应指导智能体使用 JavaScript 作为默认语言
2. 当基础提示词提及编程能力时，应明确声明 JavaScript 是优先于 Python 的首选语言
3. 基础提示词应包含清晰的指导，说明 JavaScript 是解决编程问题的默认选择

### 需求 2: 根智能体提示词更新

**用户故事:** 作为根智能体，我希望在创建子智能体时，默认的技术约束中明确使用 JavaScript，以便子智能体按照正确的语言执行任务。

#### 验收标准

1. 当根智能体为编程任务创建子智能体时，应在 taskBrief.constraints 中指定 JavaScript 为默认语言
2. 当根智能体向子智能体提供技术约束时，除非用户明确要求其他语言，否则应优先使用 JavaScript 而非 Python
3. 根智能体提示词应清楚说明 JavaScript 是所有编程相关任务的默认编程语言

### 需求 3: 工具规则提示词一致性

**用户故事:** 作为智能体，我希望工具规则提示词与默认语言设置保持一致，以便在使用工具时遵循统一的编程语言标准。

#### 验收标准

1. 当工具规则提及编程示例时，应使用 JavaScript 语法
2. 工具规则提示词应在所有编程相关指导中与默认 JavaScript 语言设置保持一致

### 需求 4: 工作空间提示词语言偏好

**用户故事:** 作为智能体，我希望工作空间提示词中的示例和指导使用 JavaScript，以便在文件操作和命令执行时保持语言一致性。

#### 验收标准

1. 当工作空间提示词提供代码示例时，应使用 JavaScript 语法
2. 当工作空间提示词描述命令执行时，应优先使用 Node.js/JavaScript 命令而非 Python 命令
